โมดูลบริหารสัญญาเช่า (Rental Contract Module)
=====================================================

## ที่มาของ requirement
ลูกค้า: บริษัท อสังหาฯ สมมุติ จำกัด (ชื่อสมมุติเพื่อฝึกอบรมเท่านั้น)
ประเภทธุรกิจ: พัฒนาอสังหาริมทรัพย์ให้เช่า — คอนโด / ร้านค้า / สำนักงาน

## ปัญหาปัจจุบัน
- ใช้ Excel ติดตามสัญญา สัญญาหลุดอายุไม่รู้ ลูกค้าต้องแจ้งเอง
- เก็บเงินประกันไว้หลายบัญชี สับสน
- Dashboard ผู้บริหารไม่มี ต้องทำ report รายเดือนด้วยมือ

## ความต้องการระบบใหม่
ลูกค้าต้องการระบบจัดการสัญญาเช่าที่ครอบคลุมตั้งแต่ร่างสัญญา ผู้เช่าเซ็น
สัญญา เก็บค่าเช่ารายเดือน แจ้งเตือนก่อนสัญญาหมดอายุ 60 วัน และสามารถต่อ
อายุสัญญาได้

### ผู้ใช้งาน 3 กลุ่ม
1) เจ้าหน้าที่การตลาด — สร้างสัญญาใหม่ บันทึกผู้เช่า
2) เจ้าหน้าที่การเงิน — ออกใบแจ้งหนี้ รับชำระ บันทึกเงินประกัน
3) ผู้บริหาร — ดู dashboard อัตราการเช่า รายได้ค้างรับ

### ข้อกำหนดทางธุรกิจ
- ผู้เช่ารายเดียวกันห้ามมีสัญญาใช้งานซ้อนกันในหน่วยเดียวกัน
- ค่าเช่าขั้นต่ำ 5,000 บาท/เดือน ขั้นสูงสุด 500,000 บาท/เดือน
- เงินประกันต้องเท่ากับค่าเช่า 2 เดือนสำหรับที่อยู่อาศัย, 3 เดือนสำหรับพาณิชย์
- ระยะเวลาสัญญาขั้นต่ำ 6 เดือน
- หน่วยเช่าที่สถานะ "กำลังซ่อมบำรุง" ห้ามทำสัญญาใหม่
- ถ้าชำระค่าเช่าช้าเกิน 15 วัน ระบบล็อคการใช้ Key Card อัตโนมัติ

### ข้อมูลที่ต้องเก็บ
- ข้อมูลผู้เช่า: ชื่อ เลขบัตร ปชช. เบอร์โทร อีเมล ที่อยู่เดิม
- ข้อมูลสัญญา: รหัสสัญญา ระยะเวลา ค่าเช่า เงินประกัน ไฟล์สัญญาสแกน
- ข้อมูลหน่วย: รหัสหน่วย ขนาด ชั้น ประเภท ค่าเช่ามาตรฐาน
- ข้อมูลการชำระ: วันที่ จำนวนเงิน วิธีชำระ สถานะ

### ความต้องการอื่นๆ (คลุมเครือ — ให้ SA จับประเด็น)
- "ระบบต้องส่งใบแจ้งหนี้ไปให้ผู้เช่าโดยอัตโนมัติ" — ไม่ระบุว่าทางไหน (email/SMS/LINE?)
- "ผู้บริหารต้องการดูรายได้แบบ real-time" — ไม่ระบุว่า real-time ระดับวินาที/นาที/ชั่วโมง?
- "ต้องรองรับการคืนเงินประกันเมื่อสัญญาสิ้นสุด" — ไม่ระบุเงื่อนไขการหัก (ค่าซ่อม?, ค่าทำความสะอาด?)
- "เชื่อมต่อกับระบบบัญชี" — ไม่ระบุว่าระบบไหน (ERP เดิม? QuickBooks? Xero?)

### ระยะเวลาและงบประมาณ
- ต้องการใช้งานภายใน 4 เดือน
- งบประมาณ 1.5 ล้านบาท
- ทีม dev ฝั่งลูกค้ามี 2 คน ฝั่ง Mango คาดว่าใช้ 3 คน

### เทคโนโลยีที่กำหนด
- Backend: Python (ลูกค้ากำหนด เพราะทีมเก่ารู้)
- Database: PostgreSQL
- Frontend: ไม่กำหนด (แนะนำได้)
- Deploy: On-premise server ของลูกค้า (Windows Server 2019)
